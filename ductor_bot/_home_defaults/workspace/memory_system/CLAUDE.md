# Memory System

This directory contains your long-term memory. `MAINMEMORY.md` persists across sessions.

## SILENCE IS MANDATORY

**Never tell the user you are reading or updating memory.** Memory operations are invisible. No announcements, no references, no "I noted that down." Just read silently, write silently, move on.

## Read First

At the start of every new session, read `MAINMEMORY.md`. It tells you who the user is and what you already know. Without it you are a stranger.

## When to Write

- User shares personal info (name, preferences, job, interests) -- write immediately, silently
- Decision made, problem solved, preference discovered -- log it
- User says "remember this" or similar -- write immediately
- New project, tool, or workflow pattern -- record it
- Cron/Webhook setup events: record inferred preference signals and likely interests (not only "created X")

## When NOT to Write

- One-time questions, throwaway tasks, debugging sessions
- Information already recorded
- Trivial or temporary context
- Every single interaction (be selective -- memory is for what matters)

## Sections

| Section | Content |
|---------|---------|
| About the User | Name, location, language, profession, personality |
| Learned Facts | Timestamped discoveries, decisions, milestones |
| Decisions and Preferences | Communication style, likes/dislikes, workflow habits, tech choices |

## Format Rules

- Keep it lean. Consolidate, merge, remove outdated entries regularly.
- Every fact gets a timestamp (`YYYY-MM-DD`).
- One line per entry. No prose, no filler.
- Use markdown sections consistently.

## Cleanup Rules

- User rejects something ("vergiss das", "stimmt nicht", "loeschen") -- remove immediately
- Outdated info (old projects, changed preferences) -- update or remove
- Duplicates -- merge into one entry
- After cleanup: no "removed" markers, just delete cleanly
